# 车间资源知识图谱系统

基于 FastAPI（后端）和 Vue 3（前端）构建的车间资源知识图谱管理系统，集成 MySQL 用户管理系统、Neo4j 图数据库知识图谱可视化、智能问答、数据导入和操作日志模块。

## 项目简介

本系统是一个车间故障诊断知识图谱平台，通过可视化展示设备、人员、工艺、物料、故障等实体之间的关系，帮助用户快速定位和解决车间设备故障。系统支持从 Excel 文件批量导入数据、可视化展示图谱、智能问答和完整的操作日志审计。

### 核心功能

- **知识图谱可视化**：交互式图谱展示，支持节点展开/折叠
- **智能检索**：支持所有字段的全文搜索（包括维护周期、出厂日期等）
- **数据导入**：支持从 Excel 文件批量导入数据（清空重建模式）
- **版本管理**：支持历史数据重新导入，实现版本回滚
- **节点分类展示**：不同类型节点使用不同颜色区分
- **动态布局**：节点展开时流畅动画，稳定后易于操作
- **用户权限管理**：基于 JWT 的身份认证和角色权限控制
- **智能问答**：基于 Qwen 大模型的 AI 故障诊断助手
- **操作日志**：完整的操作审计和日志管理

## 功能特性

### 数据导入功能（仅管理员）

- **Excel 导入**：支持从 Excel 文件批量导入数据到 Neo4j
- **清空重建**：导入前自动清空现有数据，确保数据一致性
- **宽松模式**：不限制节点类型和关系类型，自动适配 Excel 数据
- **上传记录**：保存所有上传历史，支持查看和删除
- **版本管理**：支持从历史记录重新导入，实现数据版本回滚
- **导入统计**：显示导入节点数、关系数、耗时等统计信息
- **错误报告**：详细的导入错误日志，便于排查问题

**支持的 Excel 格式**：
- 文件格式：`.xlsx`、`.xls`
- 文件大小：最大 50MB
- 必需工作表：设备台账、人员台账、工艺台账、物料台账、故障台账、三元组关系

### 知识图谱功能

- **全文搜索**：
  - 搜索所有字段属性（包括维护周期、出厂日期等）
  - 搜索关键词示例："数控"、"张三"、"焊接"、"10天"、"2024-01"
- **节点详情**：
  - 双击节点弹出详情面板
  - 显示节点所有属性信息（设备编号、型号、工位、技能等级等）
- **节点展开/折叠**：
  - 单击节点展开显示邻居节点和关系
  - 已展开节点边框加粗显示
  - 再次单击节点折叠移除之前展开的节点
  - 智能依赖清理：折叠时自动清理依赖节点
- **可视化展示**：
  - 节点显示名称而非长ID
  - 不同类型节点使用不同颜色区分
  - 关系箭头从头实体指向尾实体（有向）
  - 所有关系使用直线连接
  - 节点展开时流畅动画效果
  - 稳定后自动降低物理模拟，便于点击
  - 支持缩放、拖拽、适配视图等操作
- **图谱统计**：实时显示节点数、关系数、连接状态

### 智能问答功能

- **AI 故障诊断**：基于 Qwen 大模型的智能助手
- **对话记忆**：支持多轮对话上下文

- **实时响应**：流式输出 AI 回复
- **对话管理**：支持清除对话历史

### 操作日志功能

- **日志记录**：自动记录所有用户操作
- **统计面板**：显示总日志数、今日日志、活跃用户等
- **多维度筛选**：按用户名、行为类型、模块、状态、时间筛选
- **权限控制**：仅管理员可查看操作日志
- **分页查询**：支持大量日志数据的分页展示

### 用户管理功能

- **身份认证**：基于 JWT 的登录系统
- **用户管理**：完整的 CRUD 操作

- **状态管理**：启用/禁用用户
- **分页查询**：支持用户列表分页和筛选

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd WorkshopGraph
```

### 2. 数据库初始化

#### MySQL 初始化

```bash
# 运行 Python 脚本
cd database
python db_init.py
```

#### Neo4j 启动
```bash
# 进入 Neo4j 安装目录
cd "C:\Program Files\Neo4j\bin"

# 启动 Neo4j
neo4j.bat start-console

# 或控制台模式（前台运行）
neo4j.bat console
```

**验证 Neo4j 启动**：
- 访问：http://localhost:7474
- 使用用户名和密码登录

### 3. 后端配置

```bash
# 进入后端目录
cd backend

# 复制环境配置文件
cp .env.example .env

# 编辑 .env 文件，配置数据库连接和 API 密钥
```

`.env` 配置示例：

```env
# =========================
# MySQL 数据库配置
# =========================
DB_HOST=localhost          # MySQL 主机地址
DB_PORT=3306               # MySQL 端口
DB_USER=root               # MySQL 用户名
DB_PASSWORD=123456         # MySQL 密码
DB_NAME=workshop           # 数据库名称

# =========================
# Neo4j 图数据库配置
# =========================
NEO4J_URI=bolt://localhost:7687     # Neo4j 连接地址
NEO4J_USER=neo4j                    # Neo4j 用户名
NEO4J_PASSWORD=12345678             # Neo4j 密码
NEO4J_DATABASE=neo4j                # Neo4j 数据库名称

# =========================
# JWT 认证配置
# =========================
SECRET_KEY=your-secret-key-change-in-production    # JWT 密钥（生产环境请修改）
ALGORITHM=HS256                                    # JWT 加密算法
ACCESS_TOKEN_EXPIRE_MINUTES=1440                   # Token 有效期（分钟，默认 24 小时）

# =========================
# Qwen AI 配置
# =========================
QWEN_API_URL=https://dashscope.aliyuncs.com/compatible-mode/v1  # Qwen API 地址
QWEN_API_KEY=your-qwen-api-key-here                            # Qwen API 密钥
QWEN_MODEL=qwen-plus                                            # 模型名称

# =========================
# 前端配置
# =========================
FRONTEND_URL=http://localhost:5173    # 前端地址（用于 CORS 配置）
```

## API 接口

### 认证接口
- `POST /api/v1/auth/login` - 用户登录
- `GET /api/v1/auth/me` - 获取当前用户信息
- `POST /api/v1/auth/logout` - 用户登出

### 用户管理接口
- `GET /api/v1/users` - 获取用户列表（支持分页和筛选）
- `GET /api/v1/users/{id}` - 根据 ID 获取用户
- `POST /api/v1/users` - 创建新用户（仅管理员）
- `PUT /api/v1/users/{id}` - 更新用户信息
- `DELETE /api/v1/users/{id}` - 删除用户（仅管理员）

### 知识图谱接口
- `GET /api/v1/knowledge-graph/health` - Neo4j 健康检查
- `GET /api/v1/knowledge-graph/statistics` - 获取图谱统计信息
- `POST /api/v1/knowledge-graph/search` - 关键词搜索节点（全文搜索所有字段）
- `GET /api/v1/knowledge-graph/graph-data` - 获取图谱数据
- `GET /api/v1/knowledge-graph/neighbors/{node_id}` - 获取节点邻居（展开）

### 数据导入接口（仅管理员）
- `POST /api/v1/data-import/upload-and-import` - 上传并导入 Excel 文件
- `POST /api/v1/data-import/reimport/{file_id}` - 根据历史记录重新导入
- `GET /api/v1/data-import/records` - 获取上传记录列表
- `GET /api/v1/data-import/records/{file_id}` - 获取上传记录详情
- `DELETE /api/v1/data-import/records/{file_id}` - 删除上传记录
- `GET /api/v1/data-import/health` - 数据导入模块健康检查

### 智能问答接口
- `POST /api/v1/chat/message` - 发送消息给 AI 助手
- `POST /api/v1/chat/clear` - 清除对话历史

### 操作日志接口（仅管理员）
- `GET /api/v1/logs` - 获取操作日志列表（支持分页和筛选）
- `GET /api/v1/logs/{log_id}` - 获取操作日志详情
- `GET /api/v1/logs/statistics/summary` - 获取操作日志统计信息
- `GET /api/v1/logs/recent` - 获取最近操作日志

## 使用指南

### 数据导入页面（仅管理员）

1. 使用管理员账号登录
2. 点击导航菜单中的"数据导入"
3. **上传新文件**：
   - 点击"选择文件"按钮选择 Excel 文件
   - 点击"清空并导入"按钮执行导入
   - 等待导入完成，查看统计信息
4. **重新导入历史版本**：
   - 在"数据库上传记录"列表中选择记录
   - 点击"重新导入"按钮
   - 确认后使用该文件重新构建数据库
5. **查看记录详情**：
   - 记录列表显示文件名、大小、状态
   - 状态包括：成功、失败、导入中

**注意事项**：
- 导入操作会清空现有数据，请谨慎操作
- 建议先备份数据再进行导入
- 导入失败可查看错误信息定位问题

### 知识图谱页面

1. 登录系统后，点击导航菜单中的"知识图谱"
2. 在搜索框输入关键词进行搜索：
   - 支持搜索所有字段（包括维护周期、出厂日期等）
   - 支持中文关键词模糊匹配
   - 示例关键词："数控"、"张三"、"焊接"、"10天"、"2024-01"
3. 点击搜索按钮，系统显示匹配的节点
4. **单击节点**：展开/折叠其邻居节点
5. **双击节点**：右侧弹出详情面板，显示该节点的所有属性信息
6. **控制操作**：
   - 适配视图：自动缩放以显示所有节点
   - 放大/缩小：手动缩放图谱

### 智能问答页面

1. 点击导航菜单中的"智能问答"
2. 在输入框输入故障问题描述
3. 点击发送或按回车，AI 助手会给出诊断建议
4. 点击预设问题可快速提问

### 操作日志页面（仅管理员）

1. 使用管理员账号登录
2. 点击导航菜单中的"操作日志"
3. 查看统计信息和日志列表
4. 可按条件筛选日志

## 常见问题

### Neo4j 连接失败
1. 检查 Neo4j 是否已启动
2. 验证 `.env` 中的连接配置（host、port、用户名、密码）
3. 查看 Neo4j 日志

### 前端无法访问后端 API
1. 检查后端是否已启动（http://localhost:8000）
2. 检查浏览器控制台是否有跨域错误
3. 验证 `vite.config.js` 中的代理配置

### AI 问答无法使用
1. 检查 `.env` 中 `QWEN_API_KEY` 是否正确配置
2. 验证网络连接
3. 查看后端日志排查错误

### 数据库连接失败
1. 检查 MySQL/Neo4j 服务是否启动
2. 验证 `.env` 中的数据库配置
3. 运行 `database/db_init.py` 重新初始化数据库

### Excel 导入失败
1. 检查文件格式是否为 `.xlsx` 或 `.xls`
2. 确认文件包含必需的工作表（设备台账、人员台账、工艺台账、物料台账、故障台账、三元组关系）
3. 检查三元组关系中的实体 ID 是否在各台账中存在
4. 查看导入结果中的错误信息
5. 确认文件大小不超过 50MB

### 导入后图谱为空
1. 检查 Excel 文件中是否有数据
2. 查看导入统计信息，确认节点数大于 0
3. 检查三元组关系表是否正确填写
4. 刷新图谱页面重新加载

| 故障台账 | 故障信息 | 故障编号 |
| 三元组关系 | 实体关系 | 头实体、对象属性、尾实体 |

### 三元组关系表格式

| 列名 | 说明 | 示例 |
|------|------|------|
| 头实体 | 关系起点的实体ID | T001 |
| 对象属性 | 关系类型 | 操作设备 |
| 尾实体 | 关系终点的实体ID | M001 |

**关系示例**：
- T001（操作人员）-[操作设备]-> M001（设备）
- O001（操作人员）-[隶属]-> M002（管理人员）
- M001（设备）-[关联故障]-> F001（故障）
