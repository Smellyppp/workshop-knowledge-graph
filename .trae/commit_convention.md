# Git 提交信息规范

本项目遵循 Conventional Commits 规范，使用中文描述提交信息。

## 提交信息格式

```
<类型>: <描述>

[可选的正文]

[可选的脚注]
```

## 类型（Type）

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能或新特性 | feat: 添加用户登录功能 |
| `fix` | 修复 bug | fix: 修复用户列表分页错误 |
| `docs` | 文档或 README 更新 | docs: 更新安装说明 |
| `style` | 代码格式调整（不影响功能） | style: 统一代码缩进格式 |
| `refactor` | 代码重构（不改变功能） | refactor: 优化用户服务层代码结构 |
| `perf` | 性能优化 | perf: 优化数据库查询性能 |
| `test` | 添加或更新测试 | test: 添加用户认证单元测试 |
| `chore` | 维护任务、依赖更新等 | chore: 更新依赖包版本 |
| `ci` | CI/CD 配置变更 | ci: 添加 GitHub Actions 工作流 |
| `build` | 构建系统或外部依赖变更 | build: 更新 Vite 配置 |

## 描述（Description）

- 使用中文描述
- 简洁明了，说明本次提交做了什么
- 使用祈使句，以动词开头
- 首字母小写
- 结尾不加句号

## 示例

### 新功能
```
feat: 添加用户管理模块
```

### 修复 Bug
```
fix: 修复登录时密码验证失败的问题
```

### 文档更新
```
docs: 更新 API 接口文档
```

### 代码重构
```
refactor: 优化数据库连接池配置
```

### 依赖更新
```
chore: 升级 FastAPI 到 0.104.1 版本
```

### 性能优化
```
perf: 优化用户列表查询性能，添加索引
```

## 完整示例

```
feat: 添加用户分页查询功能

- 支持按用户名模糊查询
- 支持按用户类型筛选
- 支持按状态筛选
- 添加分页参数验证

Closes #123
```

## 注意事项

1. **提交粒度**：每次提交应该是一个完整的、可独立运行的功能或修复
2. **避免大提交**：不要在一次提交中包含多个不相关的修改
3. **清晰描述**：提交信息应该让其他人能够理解这次修改的目的
4. **关联 Issue**：如果修复了某个 issue，在脚注中使用 `Closes #123` 或 `Fixes #123` 格式
5. **禁止提交**：不要提交敏感信息（密码、密钥等）或临时文件
